
# JavaScript Scope and var vs let vs const

## ‚úÖ var vs let vs const ‚Äì Key Differences

| Feature              | `var`                       | `let`                          | `const`                        |
|----------------------|------------------------------|----------------------------------|----------------------------------|
| Scope                | Function-scoped              | Block-scoped                     | Block-scoped                     |
| Hoisting             | Yes (initialized as `undefined`) | Yes (but not initialized)        | Yes (but not initialized)        |
| Reassignment         | Yes                          | Yes                              | ‚ùå No (but objects/arrays can mutate) |
| Redeclaration        | Yes                          | ‚ùå No (in same scope)             | ‚ùå No                             |
| Temporal Dead Zone   | No                           | Yes                              | Yes                              |

---

## üß† Scope in JavaScript

### 1. Global Scope
Variables declared outside any function or block. Accessible everywhere.

```js
var x = 10;
let y = 20;
const z = 30;
```

### 2. Function Scope
`var` is limited to the function it's declared in.

```js
function example() {
    var test = "Hello";
}
console.log(test); // ‚ùå Error
```

### 3. Block Scope
`let` and `const` are limited to the block `{}` they are declared in.

```js
if (true) {
    let blockVar = 5;
    const blockConst = 10;
}
console.log(blockVar); // ‚ùå Not defined
```

---

## üß® Why is `var` avoided?

- No block scope ‚Äì leaks variables
- Can lead to bugs
- Hoisted and initialized as `undefined`
- Allows redeclaration

```js
if (true) {
    var x = 100;
}
console.log(x); // ‚úÖ 100
```

## üîç Code Breakdown Example

```js
let a = 300;
if (true) {
    let a = 10;
    const b = 20;
    var c = 30;
    console.log("INNER", a); // INNER 10
}
console.log(a); // 300
// console.log(b); ‚ùå ReferenceError
console.log(c); // ‚úÖ 30
```

- `a` inside block is a new variable
- `b` is block-scoped
- `c` is function/global scoped

## üß™ Hoisting Example

```js
console.log(x); // undefined
var x = 5;

console.log(y); // ‚ùå ReferenceError
let y = 10;
```

---

## ‚úÖ Best Practices

- Use `let` when reassignment is needed
- Use `const` by default
- Avoid `var`
- Understand block vs function scope
- Use `const` for arrays, loops like `forEach`, `map`
